# Task Progress - Fix WebSocket Disconnect & AI Not Found

## Analysis Complete - Root Causes Identified

### Bug 1 - WebSocket Disconnect
**Root Cause:** The frontend sends `{"type": "heartbeat"}` but the backend only handles `{"type": "ping"}`. The heartbeat falls through to the `else` block which sends back an error. Additionally:
- The Flutter app has NO heartbeat/ping mechanism at all
- The backend has no server-side keepalive
- The desktop app's `_scheduleReconnect` gives up after 5 attempts with no exponential backoff
- The backend's `except Exception` block closes the WebSocket with code 1011 on any error

### Bug 2 - "AI Not Found"
**Root Cause:** The backend's `ai_provider` defaults to `"openai"` (not `"ollama"`). The `.env` file doesn't set `DASH_AI_PROVIDER=ollama`. So the backend tries OpenAI, which fails because no API key is configured. Additionally:
- The model name is hardcoded as `"llama3.2"` instead of auto-detecting from Ollama
- No auto-detection of installed models via `GET /api/tags`
